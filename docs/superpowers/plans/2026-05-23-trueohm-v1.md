## Deviations from the original plan (and why)

- **Sanity checks are inline per solver, not a shared `sanity.ts`.** Each calculator's
  sanity logic is domain-specific (PF range for AC/triangle, divide-by-zero for the wheel,
  impossible-triangle, η>100%), so a shared module would have been artificial. Kept inline.
- **eslint plugins declared in root `package.json`.** The cloned TrueDrop config imports
  `typescript-eslint` / `eslint-plugin-react-hooks` / `@eslint/js` without declaring them, so
  `pnpm lint` was broken in the template. Declared them so the lint gate actually runs.
- **Additive `trueohm.css`** instead of editing the 3,200-line `global.css` — the suite CSS
  is reused verbatim; TrueOhm-only classes (scrollable 5-mode switcher, result rows, footer)
  live in a separate file.
- **iOS generation + Codemagic deferred** (TrueDrop parity — `apps/ios`/`codemagic.yaml` not
  generated in V1 core; follows the suite signing pattern when shipping to TestFlight).
